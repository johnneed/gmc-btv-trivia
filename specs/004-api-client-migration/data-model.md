### Status transition table

|---|---|---|
| `fetchQuizzes` dispatched | `"loading"` | Loader shows aria-live polite region |

| `fetchQuizzes.rejected` (401) | `"unauthorized"` | Loader shows "Sign in to view this content" message |

## New / Changed Functions

### `src/data/trivia-api.ts` (full replacement)

```
PageConfig = window.trailTriviaConfig?: { apiBase: string; nonce: string }
apiBase(): string
  Returns window.trailTriviaConfig?.apiBase ?? import.meta.env.VITE_API_BASE_URL
  Throws if both are absent/empty

fetchGames(): Promise<Quiz[]>
  GET {apiBase}/games
  AbortController: 10s timeout
  Returns: Quiz[] (published only, server-filtered)
  Throws UnauthorizedError on HTTP 401
  Throws Error on HTTP 4xx/5xx (non-401)
  Throws on timeout (AbortError ŌĆö caught by caller, shown as timeout error)

fetchGame(id: string): Promise<Quiz>
  GET {apiBase}/games/{id}
  AbortController: 10s timeout
  Returns: single Quiz
  Throws UnauthorizedError on HTTP 401
  Throws Error on HTTP 4xx/5xx

fetchAllGames(nonce: string): Promise<Quiz[]>
  GET {apiBase}/games/all
  Header: X-WP-Nonce: {nonce}
  AbortController: 10s timeout
  Returns: Quiz[] (all statuses)
  Throws UnauthorizedError on HTTP 401
  Throws Error on HTTP 4xx/5xx
```

### `UnauthorizedError` (new, in `src/data/trivia-api.ts`)

```
class UnauthorizedError extends Error {
  name = "UnauthorizedError"
}
```

Used by the thunk's `rejected` handler to detect 401 without coupling to HTTP status codes.
