# Implementation Plan: API Client Migration

**Branch**: `004-api-client-migration` | **Date**: 2026-06-26 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/004-api-client-migration/spec.md`

## Summary

Replace the static `trivia.json` fetch with three typed functions calling the WP REST API. Add `AbortController` timeouts, a distinct `"unauthorized"` Redux status, an `ErrorBoundary` with a retry button, and production-build stripping of `console.log`. The Redux layer shape changes minimally (one new status value); the React component tree gains one wrapper component. All other layers are unchanged.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|---|---|---|
| `class UnauthorizedError extends Error` (Principle I bans classes) | Typed 401 discrimination in the thunk `rejected` handler without coupling to HTTP status codes or changing the thunk's return type | `rejectWithValue` requires changing the thunk signature and breaks the `Quiz[]` return type contract; a named class `.name === "UnauthorizedError"` is the minimal discriminant that keeps the data layer pure |
