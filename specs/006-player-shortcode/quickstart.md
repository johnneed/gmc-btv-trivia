# Quickstart Validation Guide: Player Shortcode Integration (Phase 6)

Generated by `/speckit-plan` — 2026-06-26

---

## Prerequisites

- Phases 0–4 complete: plugin is activated, REST API returns games.
- Local WordPress install (6.4 or 6.5) with the trail-trivia plugin active.
- Node.js available in `react-app/` for running `npm run build:player`.
- A published game exists via the Phase 4 REST API or WP-CLI.

---

## Step 1 — Build the player bundle

```bash
cd react-app
npm run build:player
```

**Expected**: exits 0. `wp-plugin/trail-trivia/assets/player/index.js` and `index.css` exist and are non-empty.

```bash
ls -lh ../wp-plugin/trail-trivia/assets/player/
# Expected: index.js (several hundred KB) and index.css (several KB)
```

## Step 2 — PHP syntax check (SC-008 equivalent)

find wp-plugin/trail-trivia -name "*.php" -exec php -l {} \;

## Step 3 — Plugin activates cleanly

```bash
wp plugin activate trail-trivia --allow-root
echo "Exit: $?"
# Expected: 0

[ -f wp-content/debug.log ] && grep -i "fatal\|warning\|notice" wp-content/debug.log | wc -l || echo 0
# Expected: 0
```

---

## Step 4 — Create a test page with the shortcode (SC-001)

```bash
PAGE_ID=$(wp post create \
  --post_type=page \
  --post_title="Trail Trivia" \
  --post_content="[trail_trivia]" \
  --post_status=publish \
  --porcelain)

PAGE_URL=$(wp post get $PAGE_ID --field=guid)
echo "Page URL: $PAGE_URL"
```

---

## Step 5 — Verify mount point in source (SC-002)

```bash
curl -s "$PAGE_URL" | grep -c 'id="trail-trivia-root"'
# Expected: 1

curl -s "$PAGE_URL" | grep -c 'trailTriviaConfig'
# Expected: 1
```

---

## Step 6 — Verify assets loaded via WP enqueue, not hardcoded (SC-003)

```bash
# Player JS is enqueued (appears as ?ver= query param — WP enqueue pattern)
curl -s "$PAGE_URL" | grep -c 'trail-trivia-player'
# Expected: 1 (player script appears exactly once)

# No raw <script src hardcoded in PHP output
curl -s "$PAGE_URL" | grep '<script src' | grep 'assets/player' | wc -l
# Expected: 0 (WP enqueue uses id attributes, not raw src in PHP shortcode output)
```

---

## Step 7 — Verify cache-bust version in URL (SC-006a)

```bash
curl -s "$PAGE_URL" | grep 'assets/player/index.js'
# Expected: URL contains ?ver=1.0.0 (or current TRAIL_TRIVIA_VERSION)
```

## Step 9 — Browser smoke test (SC-005, SC-007)

Open `$PAGE_URL` in Chrome with the Twenty Twenty-Four theme active:

1. Open DevTools → Console tab
2. Reload the page
3. Verify: player loads and shows the game list (or empty state if no games)
4. Verify: zero `console.error` / `console.warn` messages
5. Verify: layout is not broken by theme styles
