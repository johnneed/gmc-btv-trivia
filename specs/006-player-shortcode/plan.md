# Implementation Plan: Player Shortcode Integration

**Branch**: `006-player-shortcode` | **Date**: 2026-06-26 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/006-player-shortcode/spec.md`

## Summary

Two parallel workstreams: (1) add a `build:player` Vite config that compiles the React player as an IIFE bundle into `wp-plugin/trail-trivia/assets/player/`, and (2) implement `Trail_Trivia_Shortcode` to register `[trail_trivia]`, enqueue the built assets, inject `window.trailTriviaConfig`, and render the mount point. Running `npm run build:player` then activating the plugin on any WordPress page with `[trail_trivia]` delivers the full player experience.

## Technical Context

**Language/Version**: PHP 8.0+ · TypeScript 5 · Vite 5

**Primary Dependencies**: WordPress 6.4–6.5 · `@vitejs/plugin-react` · Rollup (bundled with Vite)

**Storage**: None — Phase 6 is wiring only; no new WP options or post types

**Testing**: PHP syntax check · curl/WP-CLI integration tests · browser smoke test

**Architecture layers**: `wp-plugin/trail-trivia/includes/class-shortcode.php` (PHP) · `react-app/vite.player.config.ts` (build) · `react-app/package.json` (npm script) · `wp-plugin/trail-trivia/trail-trivia.php` (bootstrap)

## Constitution Check

- [ ] **Phase shell tests**: All MIGRATION_PLAN.md Phase 6 deterministic commands exit 0 on WP 6.4 and WP 6.5.

## Implementation Order

### Track A — Vite Build Config (no dependencies)

1. Create `react-app/vite.player.config.ts` — IIFE format, `outDir: '../wp-plugin/trail-trivia/assets/player'`, `entryFileNames: 'index.js'`, `assetFileNames: 'index[extname]'`, `emptyOutDir: true`, `esbuild.drop: ['console']`
2. Add `"build:player": "vite build --config vite.player.config.ts"` to `react-app/package.json` scripts
3. Create `wp-plugin/trail-trivia/assets/player/.gitkeep`
4. Run `npm run build:player` — exits 0; `assets/player/index.js` and `index.css` are non-empty

### Track B — PHP Shortcode (parallel with Track A)

5. Implement `Trail_Trivia_Shortcode` in `includes/class-shortcode.php`:
   - `register()` — calls `add_shortcode('trail_trivia', [$this, 'render'])`
   - `render(): string` — calls `enqueue_assets()`, returns `'<div id="trail-trivia-root"></div>'`
   - `enqueue_assets(): void` — `wp_enqueue_script('trail-trivia-player', ..., [], TRAIL_TRIVIA_VERSION, true)`, `wp_add_inline_script('trail-trivia-player', config_js, 'before')`, `wp_enqueue_style('trail-trivia-player', ..., [], TRAIL_TRIVIA_VERSION)`
6. Add `(new Trail_Trivia_Shortcode())->register()` to the existing `'init'` hook in `trail-trivia.php`
7. PHP syntax check — all plugin files pass `php -l`

### Track C — Integration Validation (requires A and B complete)

8. Create a WP test page with `[trail_trivia]`, run quickstart.md Steps 4–10
9. Run MIGRATION_PLAN.md Phase 6 deterministic test block
10. Browser smoke test — zero console errors; full quiz playthrough on WP 6.4 and 6.5

## Complexity Tracking

No constitution violations. IIFE format (research Decision 1) avoids `wp_script_add_data('type', 'module')` at the cost of bundling all dependencies into one file — acceptable for a plugin deployment model where Node.js is not available at runtime.
