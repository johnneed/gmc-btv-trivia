# Implementation Plan: Answer Image Upload

**Branch**: `009-answer-image-upload` | **Date**: 2026-06-28 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/009-answer-image-upload/spec.md`

## Summary

Replace the raw URL text field on answer image with a managed upload widget that stores images in the WordPress media library. Editors drag-and-drop or browse a local file, or paste a URL for server-side download. All images are stored as WP media attachments tagged `trivia-answer-image`. The player side is unaffected — PHP resolves attachment IDs to URLs on every GET response. TDD: tests written red-first for every new unit.

## Phase 0: Research

See [research.md](research.md). All decisions resolved:

1. Store `answerImageId` in JSON; PHP resolves to URL on read; legacy fallback to raw `answerImage` string.
2. Register `trivia_image_type` attachment taxonomy; assign `trivia-answer-image` term on upload.
3. Use `wp_remote_get()` (15s timeout) + content-type check, then `media_sideload_image()` for URL downloads.
4. Two thunks in `editor.slice.ts`; `AnswerImageUploader` is a dumb component; `GameEditor` binds callbacks.
5. `uploadingQuestionId: string | null` in `EditorState` tracks in-progress upload.

## Phase 1: Design & Contracts

### Domain type changes

**`media-attachment.type.ts`** (new):
```typescript
type MediaAttachment = {
  id: number;
  url: string;
  alt: string;
};
export default MediaAttachment;
```

**`question.type.ts`** (add one field):
```typescript
answerImageId?: number;   // WP attachment ID; 0 or absent = no managed image
```

### Admin API functions (new, in `admin-api.ts`)

```typescript
export const uploadAnswerImage = async (file: File): Promise<MediaAttachment>
export const sideloadAnswerImageFromUrl = async (url: string): Promise<MediaAttachment>
```

`uploadAnswerImage` sends `FormData` to `POST /media/upload`.  
`sideloadAnswerImageFromUrl` sends JSON to `POST /media/from-url`.  
Both return `MediaAttachment`. Errors propagate via thrown `Error` (matching existing `handleResponse` pattern).

### Editor slice additions

New state fields:
```typescript
uploadingQuestionId: string | null   // question id being uploaded; null when idle
uploadError: string | null           // error message from last failed upload; null when clear
```

New thunks:
```typescript
export const uploadQuestionImage = createAsyncThunk(
  'editor/uploadQuestionImage',
  async ({ questionId, questionIndex, file }: { questionId: string; questionIndex: number; file: File }, { dispatch, getState })
)

export const sideloadQuestionImage = createAsyncThunk(
  'editor/sideloadQuestionImage',
  async ({ questionId, questionIndex, url }: { questionId: string; questionIndex: number; url: string }, { dispatch, getState })
)
```

Each thunk:
1. Calls the corresponding `admin-api.ts` function
2. On success: dispatches `updateGameField` with updated questions array (new `answerImageId` + `answerImage` URL)
3. `extraReducers`: sets `uploadingQuestionId` to `questionId` on pending; clears to `null` on fulfilled/rejected

### QuestionCard changes

Replace:
```tsx
// REMOVE:
<label>Answer image URL <input type="url" .../></label>
<ImagePreview url={...} alt={...} />
<label>Image alt text <input .../></label>
```

Add:
```tsx
<AnswerImageUploader
  imageId={question.answerImageId}
  imageUrl={question.answerImage}
  imageAlt={question.answerImageAlt}
  isUploading={uploadingQuestionId === question.id}
  error={uploadError}
  onFileSelect={(file) => onImageUpload(question.id, index, file)}
  onUrlSubmit={(url) => onImageSideload(question.id, index, url)}
  onRemove={() => onImageRemove(index)}
  onAltChange={(alt) => onChange({ ...question, answerImageAlt: alt })}
/>
```

New props added to `QuestionCardProps`: `uploadingQuestionId: string | null`, `uploadError: string | null`, `onImageUpload`, `onImageSideload`, `onImageRemove`.

`GameEditor` binds these by dispatching `uploadQuestionImage` / `sideloadQuestionImage` thunks and passing an inline `onImageRemove` that calls `updateGameField`.

### PHP: taxonomy registration (`class-post-type.php`)

```php
define('TRAIL_TRIVIA_IMAGE_TAXONOMY', 'trivia_image_type');
define('TRAIL_TRIVIA_IMAGE_TERM',     'trivia-answer-image');

// In register() / init hook, after existing CPT:
register_taxonomy(TRAIL_TRIVIA_IMAGE_TAXONOMY, 'attachment', [
    'label'        => 'Trivia Image Type',
    'public'       => false,
    'show_ui'      => true,
    'show_in_rest' => true,
    'hierarchical' => false,
]);
```

### PHP: new REST routes (`class-rest-api.php`)

```
POST /media/upload     → handle_media_upload_handler()
POST /media/from-url   → handle_media_from_url_handler()
```

Both use `permission_callback` that calls `current_user_can('manage_trail_trivia')`.

Private helpers:
- `tag_attachment(int $id): void` — `wp_set_object_terms()` with `TRAIL_TRIVIA_IMAGE_TAXONOMY`; auto-creates term if absent
- `attachment_to_response(int $id): array` — returns `['id'=>$id, 'url'=>wp_get_attachment_image_url($id,'full'), 'alt'=>get_post_meta($id,'_wp_attachment_image_alt',true)]`

PHP resolution on GET — for each question in `build_question_response()` private helper:
```php
if (!empty($q['answerImageId'])) {
    $url = wp_get_attachment_image_url((int)$q['answerImageId'], 'full');
    $q['answerImage'] = $url ?: ($q['answerImage'] ?? '');
}
```
