`git mv`, then build the complete TypeScript domain layer (5 types, 3 factories, 1

## TDD Workflow (mandatory — Principle X)

For each factory and transform, follow this exact sequence before committing:

```
1. Write the .test.ts file (all test cases based on spec acceptance scenarios)
2. Run: npx vitest run <path/to/test-file>
3. CONFIRM all tests FAIL (red) — a passing test before implementation is a broken test
4. Write the implementation (.ts file)
5. Run: npx vitest run <path/to/test-file>
6. CONFIRM all tests PASS (green)
7. Refactor for clarity if needed — re-run to confirm still green
8. Commit test + implementation in the same commit
```

The PR description for this phase MUST include a statement confirming step 3 was observed.
