# Data Model: Monorepo Scaffold & Domain Finalization

**Date**: 2026-06-26
**Spec**: [spec.md](spec.md)

All types live in `react-app/src/domain/types/`. They are plain data structures —
no classes, no methods, no inheritance.

---

## Entity: Choice

**File**: `choice.type.ts`
**Purpose**: One answer option for a trivia question.

| Field | Type | Required | Validation | Notes |
|-------|------|----------|------------|-------|
| `id` | `string` | ✓ | UUID v4 format | Generated by `createChoice()` factory; missing from existing `trivia.json` data — factory must supply it |
| `text` | `string` | ✓ | non-empty at publish time | May be empty string during draft editing |

**Relationships**: Owned by `Question.choices[]` (always exactly 4 per question at runtime)

**State transitions**: None — `Choice` is immutable once created in the context of a published `Question`

---

## Entity: Question

**File**: `question.type.ts`
**Purpose**: A single trivia question with 4 choices, the correct answer index,
and optional explanation with image.

| Field | Type | Required | Validation | Notes |
|-------|------|----------|------------|-------|
| `id` | `string` | ✓ | UUID v4 | Generated by factory |
| `questionText` | `string` | ✓ | non-empty at publish time | Empty string allowed in draft |
| `choices` | `Choice[]` | ✓ | exactly 4 elements at publish time | Type is unbounded; publish gate enforces 4 |
| `correctAnswerIndex` | `number` | ✓ | 0–3 inclusive | Index into `choices` array |
| `answerText` | `string` | ✓ | non-empty at publish time | Explanation shown after correct answer |
| `answerImage` | `string` | — | absolute URL or empty string | Optional; shown in answer reveal |
| `answerImageAlt` | `string` | — | non-empty if `answerImage` is set | ARIA requirement for player UI |
| `answerImageCaption` | `string` | — | any string | Shown as `<figcaption>` |

**NOT included**: `tags` field — per-question tagging is not implemented (no admin UI, no REST surface)

**Relationships**: Owned by `Quiz.questions[]` (always exactly 5 per quiz at runtime)

**State transitions**: None — `Question` is a value object

## Entity: Quiz

**File**: `quiz.type.ts`
**Purpose**: A published or draft collection of exactly 5 trivia questions,
owned by a WordPress author.

| Field | Type | Required | Validation | Notes |
|-------|------|----------|------------|-------|
| `id` | `string` | ✓ | UUID v4 | Generated by factory; `_trivia_original_id` post meta for migrated games |
| `title` | `string` | ✓ | non-empty | The game's display title |
| `subtitle` | `string` | — | any string | Optional subtitle shown on game card |
| `author` | `string` | ✓ | WP display name | Denormalized for player display; not a foreign key |
| `authorId` | `number` | ✓ | WP user ID (positive integer) | Used for author display in admin UI |
| `publishDate` | `number` | ✓ | Unix ms timestamp | Stored as WP `post_date`; served as ms timestamp |
| `status` | `'draft' \| 'published'` | ✓ | enum | Maps to WP `post_status`: `draft` / `publish` |
| `questions` | `Question[]` | ✓ | exactly 5 at publish time | Type is `Question[]`; publish gate and REST enforce 5 |
| `tags` | `string[]` | ✓ | may be empty | Quiz-level tags; synced with `trivia_tag` taxonomy |

**NOT included**: `image` field — no featured image for games (editor has no image picker)

**State transitions**:
```
draft ──[publish]──► published
published ──[unpublish]──► draft
draft/published ──[trash]──► (WP trash — soft delete, recoverable)
```

**Relationships**: Has-many `Question` (owned, always 5 at runtime); belongs-to `AppUser` via `authorId`

## Entity: AppUser

**File**: `app-user.type.ts`
**Purpose**: A WordPress user as seen by the React admin UI, with role and TriviaSmith flag.

| Field | Type | Required | Validation | Notes |
|-------|------|----------|------------|-------|
| `id` | `number` | ✓ | WP user ID (positive integer) | |
| `displayName` | `string` | ✓ | non-empty | WP `display_name` |
| `roles` | `WPRole[]` | ✓ | one or more valid WP roles | `WPRole = 'contributor' \| 'author' \| 'editor' \| 'administrator'` |
| `isTriviaSmith` | `boolean` | ✓ | true iff user has `manage_trail_trivia` cap OR is admin | Single S — the canonical spelling |

**Supporting type**:
```typescript
type WPRole = 'contributor' | 'author' | 'editor' | 'administrator';
```

## Entity: PluginSettings

**File**: `plugin-settings.type.ts`
**Purpose**: User-editable plugin configuration. Only `gamesPerPage` is writable via `PUT /settings`.

| Field | Type | Required | Validation | Notes |
|-------|------|----------|------------|-------|
| `gamesPerPage` | `number` | ✓ | positive integer, default 10 | Admin game list pagination |

## Entity: PluginInfo *(read-only metadata, same file)*

**Purpose**: Read-only plugin metadata returned by `GET /settings`. Never sent in `PUT /settings`.

| Field | Type | Notes |
|-------|------|-------|
| `version` | `string` | Semver string from plugin header (e.g. `"1.0.0"`) |
| `wpMinimum` | `string` | Minimum WordPress version (e.g. `"6.4"`) |
| `phpMinimum` | `string` | Minimum PHP version (e.g. `"8.0"`) |

## Factory Defaults

| Factory | Key defaults |
|---------|-------------|
| `createChoice()` | `id: uuidv4()`, `text: ''` |
| `createQuestion()` | `id: uuidv4()`, `choices: [createChoice(), createChoice(), createChoice(), createChoice()]`, `correctAnswerIndex: 0`, `questionText: ''`, `answerText: ''` |
| `createQuiz()` | `id: uuidv4()`, `title: ''`, `status: 'draft'`, `publishDate: Date.now()`, `authorId: 0`, `author: ''`, `questions: []`, `tags: []` |

> Note: `createQuiz()` default `questions: []` is intentional — it allows incremental
> construction. The 5-question invariant is enforced at publish time by the admin UI and
> REST API, not by the factory.

## Transform: sortByDateDesc

**File**: `quiz.transforms.ts`
**Signature**: `(quizzes: Quiz[]) => Quiz[]`
**Behaviour**: Returns a new array sorted by `publishDate` descending (newest first).
**Implementation**: `R.sort(R.descend(R.prop('publishDate')))`
**Pure**: Yes — returns new array, never mutates input.
