# Quickstart Validation Guide: TriviaSmith Admin UI (Phase 5)

Generated by `/speckit-plan` — 2026-06-26

---

## Prerequisites

- Phases 0–4 complete: plugin active, REST API live.
- Node.js available in `react-app/` for `npm run build:admin`.
- Local WordPress (6.4 or 6.5) with WP-CLI.
- A user account with `manage_trail_trivia` (TriviaSmith) and one with `manage_options` (Administrator).

---

## Step 1 — Install new dependency and build admin bundle

```bash
cd react-app
npm install @dnd-kit/core @dnd-kit/sortable
npm run build:admin

ls ../wp-plugin/trail-trivia/assets/admin/
# Expected: index.js (several hundred KB) and index.css
```

## Step 3 — Activate plugin and verify admin menu (SC-002 prerequisite)

```bash
wp plugin activate trail-trivia --allow-root

# As TriviaSmith user — should see menu
wp eval "echo (int)current_user_can('manage_trail_trivia');" --allow-root
# Expected: 1 (once cap granted to a user via settings or WP-CLI)

# As non-TriviaSmith — no menu
wp eval "echo (int)user_can(CONTRIBUTOR_ID, 'manage_trail_trivia');" --allow-root
# Expected: 0
```

## Step 4 — Verify no menu for non-TriviaSmith (SC-002)

Log in to WP admin as a Contributor WITHOUT `manage_trail_trivia`. Confirm no "Trail Trivia" menu item appears in the sidebar.

## Step 5 — Create a game via the admin UI (SC-001 partial)

Log in as TriviaSmith. Navigate to Trail Trivia → All Games. Click "Add New Game".

- Fill in title and subtitle.
- Expand each of the 5 question cards, fill in question text, 4 choices, mark correct answer, fill explanation.
- Observe "Publish/Update" button is DISABLED while any question is incomplete (SC-003).
- Complete all 5 questions — observe button ENABLES (SC-003).
- Click "Save Draft" — game is saved; toolbar shows "Draft saved".
- Click "Preview Game" — modal opens showing the player UI with current data (SC-004).
- Close preview.
- Click "Publish/Update" — game is published.

Total time from "Add New Game" to Published: under 5 minutes (SC-001).

---

## Step 6 — Verify game appears in public player list

```bash
curl -s http://localhost/wp-json/trail-trivia/v1/games | jq length
# Expected: 1 (the just-published game)
```

---

## Step 7 — Test game list filters and search

On the game list:
- Click "Draft" tab — only draft games shown.
- Click "Published" tab — only published games shown.
- Type partial title in search, click Search — matching games shown.
- Clear search, click Search — full list restores.

---

## Step 8 — Test autosave (SC-005)

Open the game editor. Wait 60–62 seconds without manual save. Observe toolbar shows "Draft saved at HH:MM:SS".

---

## Step 9 — Test unpublish (no confirmation)

In the editor, click "Change" next to "Published" status. Verify status immediately changes to "Draft" with NO confirmation dialog.

---

## Step 10 — Test trash with confirmation

Click the trash icon button. Verify a confirmation dialog appears. Confirm. Game moves to trash. Verify:
```bash
curl -s http://localhost/wp-json/trail-trivia/v1/games | jq length
# Expected: 0 (trashed game absent from public list)

wp post list --post_type=trail_trivia_game --post_status=trash --format=count --allow-root
# Expected: 1 (game recoverable from trash)
```

## Step 11 — Test Settings page (SC-008)

Log in as Administrator. Navigate to Trail Trivia → Settings.

- Change `gamesPerPage` to 3, click Save. Reload — value is 3.
- Enter a valid WP username in TriviaSmith Access panel, click Grant. User appears in table.
- Enter a non-existent username, click Grant. Error message appears; no new row.
- Click Revoke on a user — they disappear from the table.
- Log in as the newly-granted user. Verify Trail Trivia menu is visible.
