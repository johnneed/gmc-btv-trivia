## Admin Redux Store Shape

The admin app has its own Redux store (separate from the player's `loader` + `score` store).

```
AdminState {
  games: GamesAdminState
  editor: EditorState
  settings: SettingsAdminState
}

GamesAdminState {
  items: Quiz[]
  total: number           // total count for pagination
  page: number            // current page (1-indexed)
  perPage: number         // from settings.gamesPerPage
  statusFilter: 'all' | 'published' | 'draft'
  searchQuery: string
  status: 'idle' | 'loading' | 'failed'
}

EditorState {
  game: Quiz | null       // the game being edited (form state)
  savedGame: Quiz | null  // last version saved to API
  isDirty: boolean        // form differs from savedGame
  autosaveStatus: 'idle' | 'saving' | 'saved' | 'failed'
  autosaveTimestamp: number | null
  publishGateOpen: boolean // true when all 5 questions complete
}

SettingsAdminState {
  gamesPerPage: number
  version: string
  wpMinimum: string
  phpMinimum: string
  TriviaSmiths: TriviaSmiths[]
  status: 'idle' | 'loading' | 'saving' | 'failed'
}

TriviaSmith {
  userId: number
  displayName: string
  roles: string[]
  isAdmin: boolean
}
```

## Admin Route Map (HashRouter)

| Hash route | Component | Requires |
|---|---|---|
| `#/` or `#/games` | `GameList` | `manage_trail_trivia` (enforced server-side) |
| `#/games/new` | `GameEditor` (blank game) | `manage_trail_trivia` |
| `#/games/:id/edit` | `GameEditor` (loaded game) | `manage_trail_trivia` |
| `#/settings` | `SettingsPage` | `manage_options` (enforced server-side) |

All three WP admin sub-pages (All Games, editor, settings) mount the same `#trail-trivia-admin-root` div; React HashRouter shows the correct view based on the hash.

## Publish Gate Logic

`publishGateOpen` is `true` when ALL of the following are true for ALL 5 questions:
- `question.questionText.trim().length > 0`
- `question.choices.length === 4`
- All 4 choices have `choice.text.trim().length > 0`

This reuses the `isComplete(question)` predicate from `src/domain/transforms/question.transforms.ts` (Phase 2).

## PHP Class: `Trail_Trivia_Admin_UI`

Location: `wp-plugin/trail-trivia/includes/class-admin-ui.php`

Responsibility: WP admin menu registration and admin asset enqueueing.

```
Methods:
  register(): void
    - Hooked to 'admin_menu'
    - add_menu_page('Trail Trivia', ..., 'manage_trail_trivia', 'trail-trivia', [$this, 'render_page'])
    - add_submenu_page('trail-trivia', 'All Games', ..., 'manage_trail_trivia', 'trail-trivia', [$this, 'render_page'])
    - add_submenu_page('trail-trivia', 'Settings', ..., 'manage_options', 'trail-trivia-settings', [$this, 'render_page'])

  render_page(): void
    - Includes templates/admin-page.php

  enqueue_scripts(string $hook_suffix): void
    - Hooked to 'admin_enqueue_scripts'
    - Returns early if $hook_suffix is not a Trail Trivia page
    - wp_enqueue_script('trail-trivia-admin', TRAIL_TRIVIA_PLUGIN_URL.'assets/admin/index.js', [], TRAIL_TRIVIA_VERSION, true)
    - wp_script_add_data('trail-trivia-admin', 'type', 'module')
    - wp_add_inline_script('trail-trivia-admin', $config_js, 'before')
    - wp_enqueue_style('trail-trivia-admin', TRAIL_TRIVIA_PLUGIN_URL.'assets/admin/index.css', [], TRAIL_TRIVIA_VERSION)
```
