## WordPress Storage

### Custom Post Type: `trail_trivia_game`

| WP field | Domain field | Notes |
|---|---|---|
| `post_title` | `Quiz.title` | `sanitize_text_field()` on write |
| `post_excerpt` | `Quiz.subtitle` | `sanitize_text_field()` on write; optional |
| `post_author` | `Quiz.authorId` | WP user ID (integer) |
| `post_status` | `Quiz.status` | `'publish'`↔`'published'`, `'draft'`↔`'draft'`, `'trash'` excluded from API |
| `post_date` | `Quiz.publishDate` | Stored as `Y-m-d H:i:s`; API converts to/from Unix ms |

### Post Meta

| Key | Type | Description |
|---|---|---|
| `_trivia_original_id` | string (UUID v4) | Canonical game identifier used as `Quiz.id`; generated on creation via `wp_generate_uuid4()` |
| `_trivia_questions` | string (JSON) | `wp_json_encode(Question[])` — exactly 5 items enforced by API |
| `_trivia_tags` | string (JSON) | `wp_json_encode(string[])` — denormalized copy of `trivia_tag` terms |
| `_trivia_schema_version` | string | Set to `"1.0"` on creation; reserved for future migrations |

### Custom Taxonomy: `trivia_tag`

- Non-hierarchical
- Attached to `trail_trivia_game` only
- Not shown in front-end nav or tag cloud
- Kept in sync with `_trivia_tags` meta on every save

### Plugin Option

| Option name | Type | Default |
|---|---|---|
| `trail_trivia_settings` | serialized PHP array | `['gamesPerPage' => 10]` |

## REST API Response Shape: `Quiz`

```json
{
  "id":           "efe84224-...",        // _trivia_original_id (UUID v4)
  "title":        "The CDT",
  "subtitle":     "Triple Crown",        // optional — empty string if absent
  "author":       "John Need",           // WP display_name of post_author
  "authorId":     1,                     // WP user ID integer
  "publishDate":  1703822400000,         // Unix milliseconds integer
  "status":       "published",           // "published" or "draft"
  "questions":    [...],                 // Question[] — always exactly 5
  "tags":         []                     // string[] — from _trivia_tags meta
}
```

### `Question` shape (within `questions` array)

```json
{
  "id":                 "6f1edeb2-...",  // UUID v4 — generated if absent on write
  "questionText":       "Which states?", // non-empty string
  "choices":            [...],           // Choice[] — always exactly 4
  "correctAnswerIndex": 1,               // integer 0–3
  "answerText":         "Because...",   // non-empty string
  "answerImage":        "https://...",  // optional string (null if absent)
  "answerImageAlt":     "Alt text",     // optional string (null if absent)
  "answerImageCaption": "Caption",      // optional string (null if absent)
}
```

### `Choice` shape (within `choices` array)

```json
{
  "id":   "3cbd3b99-...",  // UUID v4 — generated if absent on write
  "text": "Option A"       // non-empty string
}
```

## Class Responsibilities

### `Trail_Trivia_Post_Type` (`class-post-type.php`)

Responsibilities: CPT registration, taxonomy registration. Single reason to change: CPT or taxonomy definition changes.

Methods:
- `register()` — hooked to `init`, calls `register_post_type()` and `register_taxonomy()`

### `Trail_Trivia_REST_API` (`class-rest-api.php`)

Responsibilities: REST route registration and request handling. Delegates persistence to helper functions (no direct `get_post_meta()`/`update_post_meta()` calls inside route handlers — those go in private helper methods).

Methods (public): `register_routes()` hooked to `rest_api_init`
Methods (private): `get_games_handler()`, `get_game_handler()`, `get_all_games_handler()`, `create_game_handler()`, `update_game_handler()`, `patch_game_handler()`, `delete_game_handler()`, `get_settings_handler()`, `put_settings_handler()`, `build_game_response($post)`, `validate_questions($questions)`, `verify_nonce($request)`, `save_game_data($post_id, $data)`, `get_game_by_uuid($uuid)`

### `Trail_Trivia_Capabilities` (`class-capabilities.php`)

Responsibilities: Capability check helper. No WordPress hooks — pure stateless helper.

Methods: `has_manage_cap()` → `current_user_can('manage_trail_trivia')`

### `Trail_Trivia_Settings` (`class-settings.php`)

Responsibilities: Option registration and retrieval. No admin UI in Phase 4.

Methods: `register()` hooked to `init`, `get_settings()`, `get_plugin_info()`, `update_settings($data)` → `bool|WP_Error`

## `publishDate` Conversion

```
Write (Unix ms → WP datetime):
  $wp_date = date('Y-m-d H:i:s', intval($unix_ms) / 1000);

Read (WP datetime → Unix ms):
  $unix_ms = strtotime(get_post_field('post_date', $post_id)) * 1000;
```
