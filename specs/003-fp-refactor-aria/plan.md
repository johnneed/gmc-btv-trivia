# Implementation Plan: FP Refactor & ARIA Compliance

**Branch**: `003-fp-refactor-aria` | **Date**: 2026-06-26 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/003-fp-refactor-aria/spec.md`

## Summary

Phase 2 completes the structural realignment begun in Phase 0 and 1. The work has three parallel tracks:

1. **Layer cleanup**: Move the data-fetch function from `src/features/loader/loader-api.ts` to `src/data/trivia-api.ts`; extract inline transforms into `src/domain/transforms/`; delete the legacy `src/models/` directory; remove dead question-tags state from the loader slice.
2. **ARIA compliance**: Add a `ProgressBar` component, semantic question headings, `role="group"` on choice containers, `aria-live` regions for loading and score screens, `document.title` updates on route change, and framer-motion reduced-motion guards.
3. **Test coverage**: Write test files for every currently-untested `.ts`/`.tsx` file (components, features, new data and transform files) to reach ≥ 90% lines/branches across `domain/`, `store/`, `components/`, and `features/`.

The codebase is cleaner than expected: no `let`/`var` violations, no in-place mutations in source, and no component imports from forbidden layers were found in the audit. The main work is layer migration and the test suite build-out.

## Implementation Order

Tasks decompose into three tracks that can be partially parallelized. The layer cleanup must precede test writing for affected files; ARIA work is independent.

### Track A — Layer Cleanup (prerequisite for Track C)

1. Create `src/data/trivia-api.ts` (TDD: write test first → red → green)
2. Extend `src/domain/transforms/quiz.transforms.ts` with `filterPublished`, `filterByStatus` (TDD)
3. Create `src/domain/transforms/question.transforms.ts` with `isComplete` (TDD)
4. Create `src/domain/transforms/index.ts` barrel + smoke test
5. Update `loader-slice.ts`: import `fetchTrivia` from `src/data/`; remove questionTags state/actions; update type imports to `src/domain/`
6. Update `choice-button.tsx`, `carousel.tsx`, `quiz/index.tsx`: update `models/` imports → `domain/`
7. Delete `src/features/loader/loader-api.ts`
8. Delete `src/models/` directory
9. Delete `src/react-app-env.d.ts`
10. Run `npx tsc --noEmit` — must exit 0 before proceeding

### Track B — ARIA (independent)

1. Modify `carousel.tsx` — `QuestionComponent`: add `<h2 id>`, `role="group"` + `aria-labelledby`
2. Create `src/components/progress-bar/` (TDD)
3. Integrate `ProgressBar` into `quiz/index.tsx` (passes `current`, `total` props)
4. Modify `carousel.tsx` — `Carousel`: add `useReducedMotion()` guard on `motion.div`
5. Modify `home`, `quiz-list`, `score` features: add `useReducedMotion()` on their `motion.div`
6. Modify `loader/index.tsx`: wrap loading content in `aria-live="polite"` region
7. Modify `score/index.tsx`: wrap result `<p>` in `aria-live="assertive"` region
8. Add `document.title` updates in `home`, `quiz`, `quiz-list`, `score` features via `useEffect`
9. Add `<main id="main-content">` landmark in `App.tsx`; add router focus-management effect

### Track C — Test Coverage (requires Track A complete)

Write test files for all untested source files (see data-model.md for full list). All use `@testing-library/react` for components, plain Vitest for pure functions. No mocks in `src/domain/` tests.

Target: `coverage/coverage-summary.json` → `total.lines.pct >= 90`, `total.branches.pct >= 90` on `domain/`, `store/`, `components/`, `features/` layers.

### Track D — Validation

1. Run full deterministic test block from MIGRATION_PLAN.md Phase 2
2. Serve built app; run axe-core-cli; expect 0 violations
3. Manual keyboard walkthrough (see quickstart.md Step 10)
4. Manual reduced-motion check (see quickstart.md Step 11)
