## New Functions

### `src/domain/transforms/question.transforms.ts`

```
isComplete(question: Question): boolean
  Returns true when:
    - question.questionText is non-empty (length > 0 after trim)
    - question.choices has exactly 4 items
    - every choice.text is non-empty (length > 0 after trim)
  Pure function. No side effects. No mocks in tests.
  Implemented via R.all / R.pipe (Ramda).
```

### `src/domain/transforms/quiz.transforms.ts` (additions)

```
filterPublished(quizzes: Quiz[]): Quiz[]
  Returns quizzes where quiz.publishDate <= Date.now()
  Pure function. Ramda-based.

filterByStatus(status: 'draft' | 'published') => (quizzes: Quiz[]): Quiz[]
  Returns quizzes matching the given status field.
  Pure function. Ramda-based.
```

> Note: `filterValidDate` (from the old loader-api transform) is folded into `filterPublished`
> since an invalid date would never be ≤ Date.now().

### `src/domain/transforms/index.ts` (barrel)

```
Exports: { sortByDateDesc, filterPublished, filterByStatus } from quiz.transforms
Exports: { isComplete } from question.transforms
```

### `src/data/trivia-api.ts` (moved from loader-api.ts)

```
fetchTrivia(): Promise<Quiz[]>
  Fetches VITE_API_URL + "trivia.json"
  Returns raw Quiz[] (no sorting, no filtering — transforms are caller's responsibility)
  Throws if VITE_API_URL is undefined
  No console.log
```

## New Component: `ProgressBar`

Location: `src/components/progress-bar/progress-bar.tsx`

```
Props:
  current: number    // 1-indexed current question number (1–5)
  total: number      // always 5
  label?: string     // accessible label for the progress bar group, e.g. "Question 2 of 5"

Renders:
  <div role="progressbar"
       aria-valuenow={current}
       aria-valuemin={1}
       aria-valuemax={total}
       aria-label={label ?? `Question ${current} of ${total}`}>
    <div style={{ width: `${(current / total) * 100}%` }} />
  </div>

Pure dumb component. No Redux, no imports from store/data/domain.
```
